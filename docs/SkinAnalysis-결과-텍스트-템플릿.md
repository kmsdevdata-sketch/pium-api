# SkinAnalysis 결과 텍스트 템플릿 정의서

## 구조 개요

```
결과 페이지
├── 한줄요약        (ONE_LINER)         우선순위 상위 1~2개 HIGH 축 조합 ~30개
├── 레이더 차트     (7축 점수 시각화)   숫자 미노출, 상태 라벨만
├── 카테고리별 해석 (CATEGORY_TEXT)     7축 × LOW/MID/HIGH = 21개
└── 종합해석        (SUMMARY)           안전게이트 + HIGH 조합 패턴 ~20개
```

텍스트 생성 원칙
- 진단처럼 단정하지 않는다 ("~입니다" 대신 "~경향이에요", "~신호가 나타나고 있어요")
- 수치를 노출하지 않는다
- 인사이트는 "왜 이런 상태인가" + "지금 뭘 해야 하는가" 두 축으로 구성한다
- 불안감을 주는 표현을 피한다 ("심각", "위험", "문제" 사용 금지)

---

## 1. 한줄요약 (ONE_LINER)

### 생성 규칙

```
1. BARRIER HIGH or SENSITIVITY HIGH → 안전 우선 요약
2. HIGH 축 2개 이상 → 조합 요약
3. HIGH 축 1개 → 단일 축 요약
4. 전체 MID 이하 → 안정 요약
```

우선순위: BARRIER > SENSITIVITY > DRYNESS > OILINESS > BLEMISH > PIGMENTATION > AGING

---

### 안전 우선 요약 (BARRIER HIGH or SENSITIVITY HIGH 포함)

```
BARRIER_HIGH
"피부 장벽이 예민해진 상태예요. 지금은 채우는 것보다 지키는 게 먼저예요."

SENSITIVITY_HIGH
"피부가 자극에 민감하게 반응하는 시기예요. 성분보다 안정이 우선이에요."

BARRIER_HIGH + SENSITIVITY_HIGH
"장벽과 민감도 신호가 함께 높아요. 지금 피부는 쉬어야 할 타이밍이에요."

BARRIER_HIGH + DRYNESS_HIGH
"건조함과 장벽 약화가 함께 나타나고 있어요. 수분 공급보다 장벽 회복이 먼저예요."

BARRIER_HIGH + BLEMISH_HIGH
"트러블 신호는 강하지만 장벽이 먼저 안정돼야 해요. 강한 케어는 오히려 역효과일 수 있어요."

SENSITIVITY_HIGH + DRYNESS_HIGH
"건조하면서 자극에도 민감한 상태예요. 순하고 밀도 있는 보습이 지금 가장 필요해요."

SENSITIVITY_HIGH + BLEMISH_HIGH
"트러블 걱정은 크지만 지금 피부의 자극 허용도가 낮아요. 진정이 트러블 케어보다 앞서야 해요."

SENSITIVITY_HIGH + OILINESS_HIGH
"피지는 많지만 피부는 자극에 예민한 상태예요. 강한 피지 케어보다 균형 잡힌 접근이 필요해요."
```

---

### 일반 조합 요약 (HIGH 2개 이상, 안전 신호 미포함)

```
DRYNESS_HIGH + OILINESS_HIGH
"수분은 부족하고 피지는 많은 복합 상태예요. 유·수분 균형이 지금 피부의 핵심 과제예요."

DRYNESS_HIGH + BLEMISH_HIGH
"건조하면서 트러블도 반복되고 있어요. 보습 부족이 트러블을 악화시키는 패턴일 수 있어요."

DRYNESS_HIGH + AGING_HIGH
"건조함과 탄력 저하 신호가 함께 나타나고 있어요. 수분이 탄력의 기반이에요. 보습이 먼저예요."

OILINESS_HIGH + BLEMISH_HIGH
"피지 과다와 트러블 경향이 함께 강하게 나타나고 있어요. 피지 조절이 트러블 개선의 출발점이에요."

BLEMISH_HIGH + PIGMENTATION_HIGH
"트러블 흔적이 색소로 이어지는 패턴이에요. 트러블 관리가 곧 톤 케어예요."

AGING_HIGH + PIGMENTATION_HIGH
"탄력과 톤 두 가지 고민이 함께 있어요. 항산화와 재생 중심의 루틴이 두 고민을 동시에 다룰 수 있어요."

DRYNESS_HIGH + PIGMENTATION_HIGH
"건조한 피부는 톤이 더 칙칙해 보여요. 보습이 곧 미백의 첫 단계예요."
```

---

### 단일 HIGH 요약

```
DRYNESS_HIGH only
"건조 신호가 집중적으로 나타나고 있어요. 수분 공급과 유지 모두 챙겨야 할 타이밍이에요."

OILINESS_HIGH only
"피지 분비가 활발한 상태예요. 억제보다 균형 조절이 장기적으로 더 효과적이에요."

BLEMISH_HIGH only
"트러블이 반복되는 경향이 뚜렷해요. 발생 원인을 파악하는 게 케어의 시작이에요."

SENSITIVITY_HIGH only
"자극 반응이 자주 나타나는 민감한 상태예요. 새로운 제품보다 검증된 루틴이 지금은 맞아요."

PIGMENTATION_HIGH only
"톤 불균일과 색소 고민 신호가 강하게 나타나고 있어요. 자외선 차단이 모든 미백 케어의 전제예요."

AGING_HIGH only
"탄력과 주름 신호가 뚜렷하게 관측되고 있어요. 지금부터의 루틴이 5년 후 피부를 결정해요."

BARRIER_HIGH only
"피부 장벽이 약해진 신호가 나타나고 있어요. 지금은 새로운 케어를 추가하는 것보다 기본에 집중할 때예요."
```

---

### 안정 요약 (전체 MID 이하)

```
ALL_MID_OR_LOW
"전반적으로 안정적인 피부 상태예요. 지금 루틴을 유지하면서 고민 부위를 조금씩 보완해보세요."
```

## 2. 카테고리별 해석 (CATEGORY_TEXT)

각 축별 LOW/MID/HIGH 텍스트. 인사이트 = 현재 상태의 의미 + 지금 해야 할 행동 방향.

---

### DRYNESS (건조)

```
LOW
상태: "건조 불편 신호가 낮게 관측됐어요."
인사이트: "피부의 수분 유지 능력이 좋은 편이에요. 가벼운 보습으로도 충분히 유지될 가능성이 높아요.
           단, 환절기나 냉난방이 강한 환경에서는 주기적으로 상태를 확인해보세요."

MID
상태: "특정 상황에서 건조함이 반복되는 편이에요."
인사이트: "피부가 수분을 잃는 상황이 있어요. 세안 후나 건조한 환경에서 당김이 느껴진다면
           보습을 미루지 않는 습관이 중요해요. 지금 루틴이 피부를 충분히 채우고 있는지 확인해볼 시점이에요."

HIGH
상태: "당김과 건조 패턴이 뚜렷하게 나타나고 있어요."
인사이트: "단순한 수분 부족이 아니라 수분을 잡아두는 능력 자체가 약해졌을 가능성이 있어요.
           보습제의 양보다 성분이 중요한 상태예요. 세라마이드, 히알루론산처럼
           수분을 붙잡아주는 성분이 도움이 될 수 있어요."
```

---

### OILINESS (유분)

```
LOW
상태: "피지 과다 신호가 낮아요."
인사이트: "피지 분비가 적은 편이에요. 오히려 피지는 피부의 자연 보호막 역할을 해요.
           과도한 클렌징이나 피지 제거 제품은 피부 균형을 무너뜨릴 수 있으니 주의하세요."

MID
상태: "T존이나 특정 상황에서 유분 증가가 나타나는 편이에요."
인사이트: "전형적인 복합성 피부 패턴이에요. 피지가 많은 부위와 건조한 부위를 다르게 관리하는 게
           효과적이에요. 전체에 동일한 제품을 바르는 방식보다 부위별 접근을 고려해보세요."

HIGH
상태: "전반적인 피지 과다 신호가 뚜렷해요."
인사이트: "피지 분비 자체를 억제하려는 강한 케어는 피부가 더 많은 피지를 분비하는 악순환을 만들 수 있어요.
           수분 공급으로 피지 과분비의 원인을 줄이는 접근이 장기적으로 더 효과적이에요.
           논코메도제닉(모공 막힘 없는) 제품 선택이 중요해요."
```

---

### BLEMISH_PRONENESS (트러블)

```
LOW
상태: "반복 트러블 경향 신호가 낮아요."
인사이트: "지금 피부 환경이 트러블에 취약하지 않은 상태예요.
           예방 차원에서 모공을 막는 성분(코코넛 오일, 라놀린 등)이 든 제품은 피하는 게 좋아요."

MID
상태: "주기적이거나 부위성 트러블 신호가 있어요."
인사이트: "트러블 발생 패턴에 규칙성이 있을 수 있어요. 생리 주기, 스트레스, 특정 음식과의 연관성을
           체크해보세요. 부위별로 원인이 달라요. 턱 주변은 호르몬, 이마는 피지, 볼은 마찰이나 건조가 주요 원인이에요."

HIGH
상태: "반복적인 트러블 경향 신호가 강하게 나타나고 있어요."
인사이트: "트러블을 빠르게 없애려는 강한 케어가 오히려 피부 장벽을 약화시켜 트러블을 반복시킬 수 있어요.
           살리실산, 나이아신아마이드 같은 성분은 자극 없이 트러블 환경을 개선해줘요.
           트러블 케어와 장벽 케어를 동시에 진행하는 루틴이 필요한 상태예요."
```

---

### SENSITIVITY (민감성)

```
LOW
상태: "제품·환경 자극에 크게 반응하지 않는 편이에요."
인사이트: "피부 자극 허용도가 높은 편이에요. 다양한 성분 시도에 비교적 유연한 상태예요.
           다만 새로운 성분은 항상 소량 패치 테스트를 먼저 하는 습관을 유지하세요."

MID
상태: "일부 자극에서 불편 반응이 나타날 수 있어요."
인사이트: "특정 조건이나 성분에서 반응이 생기는 경향이 있어요.
           향료, 알코올, 강한 산성 성분에서 반응이 생기는 경우가 많아요.
           새 제품은 한 번에 하나씩 도입해서 어떤 성분에서 반응하는지 파악해두는 게 좋아요."

HIGH
상태: "자극 반응이 자주, 뚜렷하게 나타나고 있어요."
인사이트: "지금 피부는 많은 성분을 동시에 감당하기 어려운 상태일 수 있어요.
           루틴을 단순화하는 게 역설적으로 피부를 더 빨리 안정시켜요.
           성분 수가 적고, 향료·알코올·인공색소가 없는 제품 위주로 정리해보세요.
           레티놀, AHA/BHA 같은 강한 기능성 성분은 피부가 안정된 후에 도입하세요."
```

---

### BARRIER (장벽)

```
LOW
상태: "장벽 부담 신호가 낮아요."
인사이트: "피부 보호막이 비교적 잘 유지되고 있어요.
           과도한 각질 제거나 잦은 클렌징은 장벽을 약화시킬 수 있으니 필요 이상의 케어는 줄이세요."

MID
상태: "건조·민감 신호가 일부 겹쳐 있어요."
인사이트: "장벽이 완전히 약해진 건 아니지만 관리가 필요한 시점이에요.
           세라마이드, 판테놀처럼 장벽을 직접 채워주는 성분이 든 제품이 도움이 돼요.
           지금 쓰는 제품 중 자극이 될 수 있는 성분(알코올, 강한 향료)이 있다면 점검해보세요."

HIGH
상태: "건조와 민감 신호가 함께 높아 장벽 부담 가능성이 커요."
인사이트: "장벽이 약해지면 수분이 빠르게 날아가고, 외부 자극에도 쉽게 반응해요.
           지금은 새로운 케어를 추가하기보다 루틴을 줄이는 게 회복에 더 빠르게 작용해요.
           클렌징은 자극 없이 순하게, 보습은 세라마이드 함유 제품으로,
           기능성 성분(레티놀, 산 계열)은 장벽 회복 후로 미루세요."
```

---

### PIGMENTATION_TONE (색소·톤)

```
LOW
상태: "톤·색소 고민 신호가 낮아요."
인사이트: "현재 색소침착이나 톤 불균일 고민이 크지 않은 상태예요.
           지금 상태를 유지하는 가장 효과적인 방법은 자외선 차단이에요.
           SPF 30 이상 선크림을 매일 바르는 것만으로 색소 악화를 80% 이상 예방할 수 있어요."

MID
상태: "톤 불균일이나 잡티 고민 신호가 있어요."
인사이트: "색소침착은 자외선 노출 누적이 주원인이에요. 선케어를 가장 먼저 챙기고,
           나이아신아마이드(멜라닌 전달 억제)나 비타민C(산화 억제) 성분이 도움이 돼요.
           트러블 흔적이 색소로 이어지는 패턴이라면 트러블 케어가 미백 케어보다 우선이에요."

HIGH
상태: "톤·색소 고민 신호가 뚜렷하게 나타나고 있어요."
인사이트: "이미 생긴 색소침착은 개선에 시간이 걸려요. 빠른 효과를 기대하며 강한 성분을 시도하면
           오히려 색소가 더 짙어질 수 있어요(PIH 악화).
           선케어를 철저히 하면서 나이아신아마이드, 알부틴처럼 자극 없이 작용하는 미백 성분부터
           꾸준히 써보세요. 레티노이드 계열은 효과적이지만 피부 적응 기간이 필요해요."
```

---

### AGING_SIGNS (노화)

```
LOW
상태: "탄력·주름 고민 신호가 낮아요."
인사이트: "지금 노화 징후가 크게 느껴지지 않더라도, 예방적 루틴의 효과는 지금 시작할수록 커요.
           자외선 차단과 보습이 가장 강력한 항노화 루틴이에요. 레티놀은 지금부터 저농도로 시작하면
           나중에 더 효과적으로 활용할 수 있어요."

MID
상태: "초기 노화 고민 신호가 있어요."
인사이트: "탄력 저하와 잔주름은 피부 세포 회전 주기가 느려지면서 시작돼요.
           레티놀, 펩타이드 성분이 세포 재생을 도와줘요.
           보습이 선행되지 않으면 기능성 성분의 효과도 반감되니 루틴 순서가 중요해요."

HIGH
```

## 5. 매칭 누락 가능 지점과 fallback 전략

### 5.1 CATEGORY_TEXT는 누락이 없다

- CATEGORY_TEXT는 7축 × LOW/MID/HIGH로 이미 전부 정의되어 있다
- 따라서 각 축별 해석은 매칭이 안 될 수 없다
- 구현 시에는 각 축의 상태 라벨만 정확히 산출되면 항상 1:1 매핑된다

---

### 5.2 매칭이 안 될 수 있는 구간

- ONE_LINER 조합 요약
- SUMMARY 조합 패턴

이유:
- 현재 템플릿은 "최우선 조합" 중심으로 작성돼 있다
- 실제 결과에서는 HIGH 축이 3개 이상 동시에 뜰 수 있다
- 우선순위 상위 2개를 뽑았는데 해당 조합 문구가 없을 수 있다
- 안전 게이트는 걸렸지만, 세부 조합까지 정확히 정의되지 않은 경우가 있다
- BALANCED 모드에서 문서에 없는 2축 조합이 나올 수 있다

예상 가능한 누락 예시:
- `DRYNESS_HIGH + OILINESS_HIGH + BLEMISH_HIGH`
- `OILINESS_HIGH + AGING_HIGH`
- `BLEMISH_HIGH + AGING_HIGH + PIGMENTATION_HIGH`
- `BARRIER_HIGH + OILINESS_HIGH`
- `SENSITIVITY_HIGH + AGING_HIGH`

---

### 5.3 ONE_LINER fallback 우선순위

```
1. 안전 게이트 exact 조합 문구가 있으면 사용
2. 안전 게이트 exact 조합이 없으면 안전 단일 문구 사용
   - BARRIER_HIGH + SENSITIVITY_HIGH
   - BARRIER_HIGH only
   - SENSITIVITY_HIGH only

3. BALANCED 모드에서 상위 2개 HIGH 축 exact 조합 문구가 있으면 사용
4. exact 조합이 없으면 우선순위가 더 높은 단일 HIGH 문구 사용
5. HIGH가 없으면 ALL_MID_OR_LOW 사용
```

권장 구현 원칙:
- ONE_LINER는 "대표 메시지" 역할만 한다
- 조합 문구가 없다고 억지로 새 문장을 합성하지 않는다
- 누락 시 가장 중요한 1축 메시지로 안전하게 축소한다

---

### 5.4 SUMMARY fallback 우선순위

```
1. 안전 게이트 exact 패턴이 있으면 사용
2. 안전 게이트 exact 패턴이 없으면 안전 단일 SUMMARY 사용
   - BARRIER_HIGH only
   - SENSITIVITY_HIGH only

3. BALANCED 모드에서 상위 2개 HIGH 축 exact 조합 SUMMARY가 있으면 사용
4. exact 조합이 없으면 상위 1개 HIGH 축의 CATEGORY_TEXT(HIGH)를 기반으로 장문 확장
5. 그래도 별도 장문 템플릿이 없으면
   [상위 1개 HIGH 축 상태 문장] + [해당 축 인사이트] + [루틴 방향 1문장]
   구조로 조합
6. HIGH가 없으면 ALL_MID_OR_LOW 사용
```

권장 구현 원칙:
- SUMMARY는 조합 실패 시에도 "왜 이런 상태인지 + 지금 무엇을 해야 하는지" 두 요소를 유지한다
- 조합 문구가 없을 때는 상위 1개 HIGH 축 중심으로 축소하는 것이 가장 안정적이다
- 2개 이상의 HIGH가 있어도 문구가 없으면, 보조 축 정보는 CATEGORY_TEXT에서 보완한다

---

### 5.5 안전 게이트 fallback 세부 규칙

안전 게이트는 조합 매칭 실패보다 우선한다.

```
if BARRIER == HIGH or SENSITIVITY == HIGH:
    무조건 BARRIER_FIRST 모드
    기능성/공격적 케어보다 안정/장벽 회복 메시지를 우선 노출
```

세부 규칙:
- `BARRIER_HIGH + SENSITIVITY_HIGH` exact 문구가 최우선
- 둘 중 하나만 HIGH면 해당 단일 안전 문구 사용
- 다른 축이 함께 HIGH여도 안전 문구를 메인으로 유지
- 다른 축 정보는 CATEGORY_TEXT 또는 보조 설명에서만 반영

즉:
- `BARRIER_HIGH + OILINESS_HIGH`
  → 장벽 문구 우선, 유분 정보는 카테고리 해석에서 보완
- `SENSITIVITY_HIGH + AGING_HIGH`
  → 민감 문구 우선, 항노화 정보는 카테고리 해석에서 보완

---

### 5.6 최종 구현용 축약 규칙

```text
CATEGORY_TEXT:
항상 매칭됨 (fallback 불필요)

ONE_LINER:
안전 exact > 안전 단일 > 조합 exact > 단일 HIGH > ALL_MID_OR_LOW

SUMMARY:
안전 exact > 안전 단일 > 조합 exact > 상위 1축 장문 fallback > ALL_MID_OR_LOW
```

이 문서의 조합 템플릿은 "최우선 조합 목록"으로 사용하고,
누락 케이스는 위 fallback 순서로 처리하는 것을 기본 전략으로 한다.
