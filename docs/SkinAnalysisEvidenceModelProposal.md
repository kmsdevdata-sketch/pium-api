# SkinAnalysis 모델 근거 및 운영 원칙 (개정)

## 1. 목적
이 문서는 SkinAnalysis 모델을 근거 기반으로 운영하기 위한 기준을 정리한다.  
핵심은 특정 점수식을 고정하는 것이 아니라, 상태 모델의 타당성과 운영 검증 체계를 유지하는 것이다.

## 2. 모델링 원칙

- SkinAnalysis는 "필요 성분군" 추정보다 "현재 피부 상태(State)" 표현을 우선한다.
- 상태 축은 배타적 타입이 아니라 공존 가능한 벡터로 다룬다.
- Recommendation은 상태 벡터를 소비하되, 안전성 정책을 별도 계층에서 적용한다.

## 3. 상태 축 운영 기준

권장 상태 축:
- DRYNESS
- BARRIER
- OILINESS
- BLEMISH_PRONENESS
- SENSITIVITY
- PIGMENTATION_TONE
- AGING_SIGNS

운영 원칙:
- 축 추가/삭제는 임상 근거, 데이터 품질, 설명 가능성을 함께 검토한다.
- 축 정의는 고정 수치보다 관찰 가능한 증상/행동 기반으로 문서화한다.

## 4. BARRIER / SENSITIVITY 원칙

- BARRIER는 MVP에서 잠재 상태(latent state)로 다루고 설문 증상으로 파생 해석한다.
- SENSITIVITY/BARRIER 관련 신호는 추천에서 효능 가산보다 안전성 제한에 우선 사용한다.
- 구체 계수/임계값은 문서 고정 대신 `rulesVersion`으로 관리한다.

## 5. 설문-모델 연계 원칙

- 설문 문항은 상태 추정 품질을 높이는 방향으로 설계한다.
- 문항 변경이 잦을 수 있으므로, 질문-지표 매핑은 설정 데이터와 버전으로 분리한다.
- "잘 모르겠음" 응답과 무성의 응답을 처리하는 방어 로직을 포함한다.

## 6. 검증과 개선

- 운영 지표:
  - 설문 완료율
  - 문항별 응답 분산/일관성
  - 추천 만족도 및 부정 피드백율
  - 안전 관련 불만/중단 이벤트
- 개선 방식:
  - 정책 버전 단위 A/B 테스트
  - 문항 집합/순서/표현의 점진적 개선
  - 설명문 품질(이해도/수용성) 점검

## 7. 결론

본 프로젝트는 다음 기준을 따른다.
- State-based 모델 우선
- Safety-first 추천 정책
- 고정 점수식 배제, 버전 기반 지속 튜닝

이 문서는 상세 계수표가 아니라, 모델 품질을 유지하기 위한 운영 가이드로 사용한다.
