# SkinAnalysis 해석 모델 운영 원칙 (MVP)

## 1. 목적
이 문서는 SkinAnalysis 결과를 "점수"가 아닌 "상태"로 해석하기 위한 MVP 운영 기준을 정리한다.
핵심은 과도한 정밀 점수 해석을 피하고, 사용자에게 이해 가능한 상태 설명을 제공하는 것이다.

## 2. 현재 문제 정의
- 설문 점수는 물리 단위가 아니므로 `70 vs 71`을 임상적 차이로 해석하기 어렵다.
- `10 vs 70`을 비율(7배)로 해석하면 과대해석 위험이 있다.
- 단일 축 중심 해석은 실제 피부의 중첩 상태를 설명하지 못한다.

## 3. 해석 철학
- 점수는 내부 계산 신호로만 사용한다.
- 사용자에게는 점수 수치 텍스트 대신 상태 라벨과 근거 문장으로 전달한다.
- 단, 7축 상태 그래프(레이더/폴리곤) 시각화에는 내부 점수를 사용한다.
- `HIGH`는 "나쁨"이 아니라 "해당 상태 신호가 강하게 관측됨"을 의미한다.
- 단일 축 우선순위가 아니라 축 조합 기반으로 해석한다.

## 4. 상태축 해석 기준 (MVP)
모든 상태축은 `LOW / MID / HIGH` 3단계로만 해석한다.

- DRYNESS
  - LOW: 건조 불편 신호가 낮음
  - MID: 특정 상황에서 건조감이 반복됨
  - HIGH: 당김/건조 패턴이 뚜렷하게 반복됨
- OILINESS
  - LOW: 피지 과다 신호가 낮음
  - MID: T존/상황성 유분 증가
  - HIGH: 전반적 번들/피지 과다 패턴이 뚜렷함
- BLEMISH_PRONENESS
  - LOW: 반복 트러블 경향 신호가 낮음
  - MID: 주기적/부위성 트러블 신호가 있음
  - HIGH: 반복적 트러블 경향 신호가 강함
- SENSITIVITY
  - LOW: 제품/환경 자극 반응 신호가 낮음
  - MID: 일부 자극에서 불편 반응 가능
  - HIGH: 자극 반응이 자주/뚜렷하게 관측됨
- BARRIER (파생)
  - LOW: 장벽 부담 가능성 신호가 낮음
  - MID: 건조/민감 신호가 일부 중첩됨
  - HIGH: 건조/민감 신호가 함께 높아 장벽 부담 가능성이 큼
- PIGMENTATION_TONE
  - LOW: 톤/색소 고민 신호가 낮음
  - MID: 톤 불균일/잡티 고민 신호가 있음
  - HIGH: 톤/색소 고민 신호가 뚜렷함
- AGING_SIGNS
  - LOW: 탄력/주름 고민 신호가 낮음
  - MID: 초기 노화 고민 신호가 있음
  - HIGH: 탄력/주름 고민 신호가 뚜렷함

## 5. BARRIER 정의
- BARRIER는 직접 문항으로 측정하지 않는다.
- DRYNESS, SENSITIVITY, 응답 패턴에서 파생되는 "안전 판단용 신호"로 해석한다.

## 6. 조합 해석 원칙
다음은 대표 조합 해석 예시다.

- DRYNESS HIGH + OILINESS HIGH: 수분 부족과 피지 과다가 공존하는 복합 상태
- BLEMISH HIGH + SENSITIVITY HIGH: 트러블 신호는 높지만 자극 허용도는 낮은 상태
- AGING HIGH + BARRIER HIGH: 탄력 고민 신호는 크지만 장벽 안정이 선행되어야 하는 상태

## 7. 사용자 응답 원칙
- 숫자 점수는 노출하지 않는다.
- 질환 진단처럼 단정하지 않는다.
- "무엇이 관측되었는지" 중심으로 간결하게 설명한다.
- 축별로 짧은 근거 문장을 1~2개 제공한다.

## 8. 결론
SkinAnalysis의 핵심은 점수 산출이 아니라 상태 해석이다.
MVP에서는 3단계 라벨과 조합 해석으로 운영하고, 안전성이 필요한 경우 보수적 추천 모드로 연결한다.
내부 점수는 7축 상태 그래프 시각화와 추천 매핑 연산에 사용하되, 사용자 커뮤니케이션은 상태 라벨/근거 문장 중심으로 유지한다.
